# v1.0.3 — README audit fixes

**Doc-only patch.** No behavioral change. No new features, no new
migrations, no new validator logic. Brings the README into line with the
v1.0.0 / v1.0.1 / v1.0.2 state of the repo.

## What changed

The README was silently stale in three high-traffic areas:

- **Versioning section** described the v1.0.0 default-install behavior
  ("pulls `main` (bleeding edge)") instead of the v1.0.1 default
  ("pulls `v1` (latest v1.x.x)"). Updated. Also added a one-liner
  teaching users how to check their currently installed version
  (`cat <skill_dir>/VERSION`).
- **Migrations section** did not exist. The README is the user-facing
  install/setup doc, and it had zero mention of the migration registry,
  the `migrate.mjs` runner, the registered migrations, or the
  per-project ledger. Added a `## Migrations` section listing the two
  currently registered migrations, the runner CLI flags
  (`--list`, `--dry-run`, `--project`, `--pm-folder`, `--migration <id>`,
  `--yes`), and the `<pm_folder>/.pm/migrations.json` ledger location.
- **Repository Map** did not list `scripts/migrate.mjs`,
  `scripts/migrations/`, or `docs/releases/`. Added all three.

Two smaller fixes folded in:

- The validator table row for `check-vault-structure.mjs` did not mention
  migration detection; updated to note that it emits
  `## Unapplied Migrations` for the migration registry.
- The `decisions/` lane row in the PM Folder Model did not list the
  seven decision type codes; updated to include
  `ADR / PRD / MKT / VND / POL / NEG / EXP`.

## Who it affects

Anyone reading the README — fresh users following the install path, or
existing users who hit a `## Unapplied Migrations` warning and want to
know what migrations are and how to run them.

## What it does NOT change

- No new migration. If you already ran v1.0.2 against your projects,
  you don't need to re-run anything.
- No validator logic changed. The validators are still the same; their
  *output* is the same; their *behavior* is the same.
- The CHANGELOG and release notes for v1.0.0, v1.0.1, and v1.0.2 are
  unchanged (those are historical records of what shipped in those
  releases; they are not edited retroactively).

## Install / upgrade

```bash
# Fresh install on the v1 channel (default):
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --target agents --yes

# Update an existing install:
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --target agents --yes --update

# Pin to v1.0.3 exactly:
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1.0.3/install.sh | bash -s -- --target agents --yes
```

After install, verify the version with `cat <skill_dir>/VERSION` (should
read `1.0.3`).
