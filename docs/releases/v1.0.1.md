# v1.0.1 — Flip install defaults to v1 channel

**Behavior change for fresh installs only.** Existing installs are
unaffected. The default install channel now resolves to `v1` (latest
v1.x.x release) instead of `main` (bleeding edge).

## What's changed

- **`install.sh` default `--ref` is now `v1`.** Fresh installs and
  `install.sh --update` calls resolve to the latest `v1.x.x` tag.
- **`openclaw-instruction.md` install URLs use `v1`.** OpenClaw PM
  bootstrap installs the latest stable release by default.
- **The bleeding edge is still available.** Pass `--ref main` or
  `--channel main` to install the latest commit on `main`.

## Who it affects

- **Fresh installs (no existing project-management skill on disk):** get
  `v1` instead of `main`. No action required; this is what most users want.
- **Existing installs (already on `main`):** continue tracking `main` until
  you re-run `install.sh` without `--ref`. To move to `v1` after this
  release, run `curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --update` (or `--target agents`, etc.).
- **Users on a pinned version (`--ref v1.0.0`):** unaffected. Pinned
  installs always resolve to the exact tag.

## Why

Before this release, `install.sh` always resolved to `main`, which meant
every commit on `main` was a candidate for fresh installs and `update.sh`.
After the `v1.0.0` migration registry landed, we needed a stable channel
so users running `update.sh` would not pick up breaking changes
mid-session. The `v1` tag is a moving target that always points at the
latest `v1.x.x` release; bleeding-edge users can opt in with `--ref main`.

## Migration note

The `1.0.0-lane-restructure` migration has been part of the registry
since `v1.0.0`. No new migrations in this release. Existing users who
have not yet migrated will see the same `## Unapplied Migrations`
section in their validator output as before.

## Install / upgrade

```bash
# Fresh install on the v1 channel (default):
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --target agents --yes

# Update an existing install to the latest v1.x.x:
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --target agents --yes --update

# Pin to v1.0.1 exactly:
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1.0.1/install.sh | bash -s -- --target agents --yes

# Bleeding edge (latest commit on main):
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/main/install.sh | bash -s -- --target agents --yes
```
