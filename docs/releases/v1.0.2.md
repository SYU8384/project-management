# v1.0.2 — Rewrite stale v0.x body text after v1.0.0 migration

**Bug-fix release.** No new features. The v1.0.0 lane restructure succeeded at
folder moves and wikilink rewrites but missed the prose-level cleanup. This
release adds a registered migration that catches up the rest.

## What's in this release

A second registered migration, `1.0.2-v0-content-rewrite`, registered in
the same migration registry as `1.0.0-lane-restructure`. It is
**idempotent** — running it on a project that has already been migrated to
v1.0.2 conventions is a no-op.

The migration auto-rewrites:

- **`decisions/decisions.md`**: replace v0.x ADR-monoculture intro with v1.0.0
  typed-decision language; drop the v0.x `## Decisions Log` section; drop
  the v0.x `planning/decisions` alias from frontmatter.
- **`roadmap/plans/plans.md`**: H1 `# planning` → `# plans`, title
  `planning` → `plans`, add a `## Conventions` section covering filename,
  H1, status values, archived field, cross-link, and "Decisions cited, not
  duplicated" rules.
- **`archive/archive.md`**: replace "or planning docs." with the v1.0.0 lane
  naming.
- **`## Relevant ADRs` → `## Relevant Decisions`** and **`> No ADRs yet.` →
  `> No decisions yet.`** in any project file.
- **`templates/ADR.md` → `templates/decision.md`** in any project file.
- **Frontmatter fields**: `current_behavior_source`, `source_of_truth`,
  `related` — `planning/` → `roadmap/plans/` (preserves inline-list and
  block-list YAML styles).
- **v0.x tags**: drop `wip` and `deprecated` from non-archive files (drops
  the entire line if the tag list becomes empty).
- **`status: in-progress` → `status: active`** (the v1.0.0 enum doesn't
  include `in-progress`; this is the conservative fix).
- **Decision file body shape**: `## Implementation Notes` →
  `## Realization Notes`; `## Alternatives considered` →
  `## Options Considered`.
- **Decision file title and H1**: `ADR-NNN: …` → `D-NNN: …` (the file was
  already renamed to `D-NNN_ADR_*.md` by v1.0.0; this catches up the
  in-file frontmatter and H1).
- **Plan H1**: rewrite to slug-only (matching filename stem); preserve the
  original descriptive title as a `## Summary` section if not already
  present.
- **Broken wikilinks**: lines with `[[` but no matching `]]` get the
  missing `]]` appended.
- **`roadmap/done-pending.md`**: date-prefixed section headers
  `## YYYY-MM-DD_slug` → `## slug` (mirrors the v1.0.0 plan-naming rule).

The migration surfaces the following items as **MANUAL REVIEW** warnings
(not auto-fixed — they need human judgement):

- Plan status/body mismatches: frontmatter `status: active` but body says
  "SUPERSEDED by …" / "NOT YET APPROVED" / "DRAFT".
- Decision content authoring: plans with `## Decisions Locked` or
  `## Decisions (all explicit, …)` sections that duplicate
  decision-record content; the v1.0.0 rule is "Decisions cited, not
  duplicated," which means splitting these into typed
  `decisions/D-NNN_<type>_<slug>.md` files.
- Known-issues theoretical-risk wording: lines in `roadmap/known-issues.md`
  that say "possible" / "could" / "if X happens" without observed
  evidence.

The validator (`check-vault-structure.mjs`) automatically surfaces
`1.0.2-v0-content-rewrite` as an `## Unapplied Migrations` section for any
project where `detect()` returns true.

## Who it affects

**Anyone who ran the v1.0.0 lane restructure on a project before this
release.** Your project still has v0.x prose in folder notes, README
tables, feature pages, plan H1s, decision title/H1s, and frontmatter
fields. The agent will offer to run this migration on your next PM-skill
interaction. You can also run it manually:

```bash
node <skill_dir>/scripts/migrate.mjs --project <Name> --yes
```

Add `--dry-run` first to see the plan. Add `--list` to see all registered
migrations.

**Anyone with a fresh project on v1.0.2**: nothing to do. The new
`bootstrap-pm.mjs` already scaffolds v1.0.2-conformant content; no
migration will fire.

## What it does NOT do

- It does NOT split inline decisions into D-NNN files. That requires
  content authoring (turning a paragraph in a plan into a structured
  decision record). Surfaced as MANUAL REVIEW.
- It does NOT touch `archive/` or `history/` content. Those are immutable
  by design.
- It does NOT rewrite the body of feature pages that were copy-pasted
  from a different project (the audit found this on Odyssey's
  `features/decisions.md` and `features/market-research.md`).
- It does NOT address the Syncthing `.stversions/` snapshot directory
  stale files (out of scope; Syncthing-managed).

```bash
# Fresh install on the v1 channel (default):
curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --target agents --yes

curl -fsSL https://raw.githubusercontent.com/SYU8384/project-management/v1/install.sh | bash -s -- --target agents --yes --update

node <skill_dir>/scripts/migrate.mjs --project <Name> --yes
```
